# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A local-first personal knowledge base ("second brain") that ingests PDFs, converts them to Markdown, builds a graph knowledge structure, and answers questions by querying local data first — escalating to the Claude API only when local context is insufficient. The UI mimics an 80s home computer terminal (web-based, phosphor aesthetic).

---

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Start the full application (watcher + web server)
python -m brain.app

# Run the ingestion pipeline on a specific file
python -m brain.ingest path/to/file.pdf

# Process all pending files in raw/
python -m brain.ingest --all

# Start only the web UI
python -m brain.server

# Run tests
pytest

# Run a single test file
pytest tests/test_parser.py

# Run tests matching a pattern
pytest -k "test_pdf"

# Rebuild the vector index from scratch
python -m brain.index --rebuild

# Export the knowledge graph as JSON
python -m brain.graph export graph.json
```

## Architecture

### Directory Layout

```
second-brain/
├── raw/                  # Drop PDFs here; watcher picks them up automatically
├── brain/
│   ├── app.py            # Entry point: starts watcher + FastAPI server
│   ├── server.py         # FastAPI routes and MCP server
│   ├── ingest.py         # Orchestrates the full ingestion pipeline
│   ├── parser.py         # PDF → Markdown conversion (docling/pymupdf4llm)
│   ├── vision.py         # Claude vision API calls for image description
│   ├── tagger.py         # Auto-tagging via Claude API (skipped if user provides tags)
│   ├── linker.py         # Wikilink generation between documents
│   ├── graph.py          # Graph construction, traversal, persistence (NetworkX + SQLite)
│   ├── index.py          # Vector index management (ChromaDB)
│   ├── search.py         # Hybrid search: BM25 + vector + graph traversal
│   ├── query.py          # Q&A engine: local-first, Claude API escalation
│   └── mcp.py            # MCP server exposing brain tools
├── store/
│   ├── documents/        # Processed Markdown files
│   ├── images/           # Extracted images from PDFs
│   ├── graph.db          # SQLite: graph edges, document metadata, tags
│   └── vectors/          # ChromaDB persistent storage
├── static/               # Retro CSS, JS, fonts (VT323 / Share Tech Mono)
├── templates/            # Jinja2 HTML templates
├── tests/
└── requirements.txt
```

### Ingestion Pipeline

```
raw/*.pdf
  → parser.py       PDF → Markdown + extract images to store/images/
  → vision.py       each image → Claude vision → description injected into Markdown
  → tagger.py       if no manual tags → Claude API generates tags
  → linker.py       scan existing documents → generate [[wikilinks]]
  → index.py        chunk + embed → ChromaDB
  → graph.py        upsert node + edges (wikilinks, shared tags, semantic similarity)
  → store/documents/
```

### Query Flow

```
User question
  → search.py      BM25 keyword pass (rank_bm25)
  → search.py      vector similarity pass (ChromaDB)
  → graph.py       graph traversal from top hits (1-2 hops via wikilinks)
  → query.py       assemble context window from ranked chunks
  → confidence check: if context score > threshold → answer locally
                   else → Claude API with local context as grounding
  → response + cited sources
```

### Graph RAG

- **Nodes**: one per processed document (stored in SQLite `documents` table)
- **Edges**: three types stored in SQLite `edges` table:
  - `wikilink` — explicit `[[doc]]` references generated by `linker.py`
  - `tag` — documents sharing a tag
  - `semantic` — cosine similarity above 0.85 (built at index time)
- **In-memory graph**: NetworkX `DiGraph`, rebuilt from SQLite on startup
- **Visualization**: Cytoscape.js graph rendered in the web UI; nodes glow on hover, edges colored by type

### Claude API Usage

| Module | Model | Purpose |
|---|---|---|
| `vision.py` | `claude-haiku-4-5` | Image descriptions (cost-sensitive, high volume) |
| `tagger.py` | `claude-haiku-4-5` | Auto-tag generation |
| `linker.py` | `claude-sonnet-4-6` | Wikilink suggestions (needs reasoning) |
| `query.py` | `claude-sonnet-4-6` | Q&A escalation fallback |

Always use the `anthropic` Python SDK with prompt caching enabled on long system prompts and document context.

### MCP Server (`brain/mcp.py`)

Exposes the brain as an MCP server so Claude Code (or any MCP client) can query the brain directly. Tools to implement:

- `search(query, limit)` — hybrid search
- `get_document(slug)` — retrieve Markdown content
- `get_graph_neighbors(slug, hops)` — graph traversal
- `ingest(file_path)` — trigger ingestion pipeline
- `list_tags()` — all tags in the brain

### Web UI

- **Framework**: FastAPI + Jinja2 templates
- **Aesthetic**: Terminal CLI — phosphor green (`#33ff33`) on OLED black (`#050505`); amber (`#ffb000`) accents; subtle CRT scanline overlay (CSS `::after`, 0.07 opacity); JetBrains Mono as primary font, VT323 for display headers
- **Design tokens**: all colors, spacing (4px base), typography scale, animation durations, and glow values live in CSS custom properties on `:root` in `static/style.css`
- **Accessibility**: skip link, ARIA landmarks (`role="banner/navigation/main"`), `aria-current="page"` on active nav link, `aria-live="polite"` on terminal output and upload status, `prefers-reduced-motion` respected for all animations, visible focus rings (2px solid `--fg`)
- **Responsive**: mobile-first; hamburger nav below 640px; font sizes and padding adjust at breakpoint
- **Pages**:
  - `/` — search/Q&A terminal prompt; `role="log"` output div; command history (↑/↓); Escape to clear; typewriter answer reveal; safe DOM construction (no `innerHTML` for user-influenced content)
  - `/graph` — Cytoscape.js knowledge graph; edge type coloring (wikilink=green, tag=amber, semantic=cyan); node search filter; zoom controls; node info sidebar; graph legend; empty state when no docs indexed; full-bleed layout (`main.full-bleed`)
  - `/doc/<slug>` — rendered Markdown; breadcrumb nav (`← TERMINAL / GRAPH / slug`); `article` landmark
  - `/upload` — drag-and-drop with animated progress bar; ARIA-labelled drop zone; upload state classes (`success`/`error`/`info`)

## Conventions

- **Slug**: document identifier derived from filename, snake_case, no extension (e.g. `quantum_computing_intro`)
- **Wikilinks**: always `[[slug]]` format, never display text aliases — keep links machine-parseable
- **Manual tags**: stored in Markdown frontmatter (`tags: [foo, bar]`); if frontmatter has `tags`, `tagger.py` skips LLM call entirely
- **Confidence threshold**: configurable via `QUERY_CONFIDENCE_THRESHOLD` env var (default `0.72`); below this, escalate to Claude API
- **Images**: stored as `store/images/<slug>/<n>.png`; Markdown references them as relative paths; vision description injected as a blockquote immediately after the image link
- **Graph DB schema**: keep migrations in `brain/migrations/`; apply with `python -m brain.migrations`

## Environment Variables

```
ANTHROPIC_API_KEY        required — Claude API key
BRAIN_ROOT               optional — path to store/ directory (default: ./store)
QUERY_CONFIDENCE_THRESHOLD  optional — float 0-1 (default: 0.72)
SERVER_PORT              optional — web server port (default: 8000)
```
